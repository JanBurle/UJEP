## Princip OOP

OOP je zaměřeno na _objekty_ a jejich vzájemné interakce.
Objekt je _instance třídy_.

- Objekt: předmět, věc, entita, ...
- Instance: výskyt (prvku), případ, obraz, podoba, ...
- Třída: kategorie, skupina, společenství, seskupení, ...

### Čtyři základní principy:

- **zapouzdření** (encapsulation, data hiding): skryté vnitřní
  detaily dat a implementace, veřejné rozhraní (interface) –
  metody
- **dědičnost** (inheritance): vytváření nových tříd jako
  specializace (subclass, podtřída) na základě existujících
  tříd (superclass, nadřazená třída), z kterých se dědí
  vlastnosti a chování
- **polymorfismus**, mnohotvárnost (polymorphism): stejné
  rozhraní (interface) pro různé třídy, ale různé chování
  (implementace), přepisování a přetěžování (overriding a
  overloading) metod
- **abstrakce**: definice obecných tříd a jejich rozhraní, pak
  různé implementace, umožňuje "pohled z výšky" (bird's eye),
  bez zabývání se vnitřními detaily

#### Intuitivní vysvětlení:

Třídy jsou šablony, podle kterých se vytvářejí objekty.
Objekty jsou "chytrá data": data a metody, které s nimi
pracují, nebo také data, která se umí samostatně zpracovávat.

### Funkce a metody – rozdíly mezi nimi

Metoda třídy je v zásadě funkce se skrytým (prvním)
parametrem, kterým je objekt (instance třídy), nad kterým je
metoda volána. Tento parametr se v Pythonu obvykle nazývá
`self` (v jiných jazycích `this`). Místo zadávání tohoto
parametru při volání metody je tento automaticky předán. Při
definici metody je ale potřeba tento parametr uvést.

Příklad:

```python
lst = [2,3,1,4]

sorted(lst)      # funkce
lst.sort()       # metoda
```

Poznámka: kromě automatického předávání parametru `self`, mají
metody vlastnost "override" (přepisování), což znamená, že
můžeme definovat metodu se stejným názvem jaký má metoda v
nadřazené třídě a tím předefinovat chování této metody.

## OOP v Pythonu

Python je plně OO. Uživatelské třídy lze definovat nebo dědit
ze zabudovaných (vestavěných) tříd.

_Třída_ (class) a _typ_ (type) jsou v Pythonu synonyma. Místo
_třída_ lze říct _typ objektu_.

Typ objektu je uložen v atributu `__class__` objektu.
("Double-underscore", "dunder" atributy `__xxx__` jsou
"magické".)

```python
x = 42
print(x.__class__) # <class 'int'>
```

Magičnost atributů `__xxx__` je v tom, že je Python volá
automaticky, když je potřeba. Např. atribut `__class__` je
volán funkcí `type()`.

```python
x = 42
print(type(x))  # <class 'int'>

y = "Hello"
print(type(y))  # <class 'str'>

z = [1, 2, 3]
print(type(z))  # <class 'list'>
```

### Výpis atributů a metod objektu nebo třídy

Vestavěná funkce `dir()` vypíše atributy a metody objektu nebo
třídy.

```python
x = 42
print(dir(x))
print(dir(type(x)))
```

Samotná třída (objekt) je instance třídy `type`. Má atribut
`__name__`:

```python
x = 42
print(x.__class__)          # <class 'int'>
print(x.__class__.__name__) # int
```

## Definice uživatelských tříd

Jednoduchá definice prázdné třídy:

```python
class Dog:              # Definice třídy (hlavička)
  pass                  # Tělo definice třídy (prázdné)

d = Dog()               # Vytvoření instance třídy

print(type(d))          # <class '__main__.Dog'>
print(d.__class__)      # <class '__main__.Dog'>
print(type(d).__name__) # Dog
```

Třídy mohou být lokální:

```python
def foo():
  class Cat:
    pass
  c = Cat()
  print(type(c))

foo()

cc = Cat()        # NameError: name 'Cat' is not defined
```

Magická metoda `__init__` je _konstruktor_ třídy. Je volána
automaticky při vytváření instance.

```python
class Dog:
  def __init__(self, name: str):  # construktor
    self.name = name              # instance attribute (member/členský atribut)

  def bark(self):
    print(f'Haf! Jsem {self.name}!')

raf = Dog("Rafík")
ram = Dog("Rampík")

raf.bark()
ram.bark()

dogs = [raf, ram]
for d in dogs:
  d.bark()
```

## Hierarchie tříd

Dědičnost (inheritance) je základním prvkem OOP. Třída může
dědit od jiné třídy. Základní třída všech objektů v Pythonu je
třída `object`.

```python
class Aardvark:
  pass

print(Aardvark.__bases__)       # (<class 'object'>,)
print((9).__class__.__bases__)  # (<class 'object'>,)
print(object.__bases__)         # ()
```

Vlastní hierarchie:

```python
class Thing:
  pass

class Machine(Thing):
  pass

class LivingThing(Thing):
  pass

class Robot(Machine):
  pass

class Animal(LivingThing):
  pass

class Cyborg(Animal, Robot):
  pass

class Mammal(LivingThing):
  pass

class Dog(Mammal):
  pass

class Cat(Mammal):
  pass

# bases
print(Cyborg.__bases__)
print(Robot.__bases__)
print(Machine.__bases__)
print(Thing.__bases__)

# subclasses
print(Thing.__subclasses__())
```

Vypsat hierarchii podtříd:

```python
def print_cls_tree(cls: type, level=0):
  prefix = '  ' * level + '└── ' if 0 < level else ''
  print(prefix + cls.__name__)
  for subclass in cls.__subclasses__():
    print_cls_tree(subclass, level + 1)

# Start with the base class
print_cls_tree(Thing)
```

Vestavěná hierarchie:

```python
print_cls_tree(int)
```

## Zděděné atributy a metody

V poslední hierarchii jsme ztratili možnost pojmenovat psa, a
pes také už neumí štěkat:

```python
ralf = Dog('Ralf')
# ralf = Dog()
ralf.bark()
```

Místo toho pojmenujeme všechny věci tím, že jméno nadefinujeme
v základní třídě. Dále přidáme metody `sound()` a `hello()`.
Metodu `sound()` přepíšeme v některých podtřídách.

```python
class Thing:
  def __init__(self, name: str):
    self.name = name

  def sound(self):
    return ''

  def hello(self):
    print(f'Hi, I am {self.name} - {self.sound()}!')

class Machine(Thing):
  def sound(self):
    return 'wrrr'

class LivingThing(Thing):
  pass

class Robot(Machine):
  pass

class Animal(LivingThing):
  pass

class Cyborg(Animal, Robot):
  pass

class Mammal(LivingThing):
  pass

class Dog(Mammal):
  def sound(self):
    return 'woof'

class Cat(Mammal):
  def sound(self):
    return 'miaow'

Thing('thing').hello()
Robot('Marvin').hello()
Dog('Ralf').hello()
Cat('Mieze').hello() # Miezekatze - micka
```

## `__repr__` a `__str__`

Vytiskneme micku (převedeme ji na řetězec):

```python
cat = Cat('Mieze')
cat.hello()
print(cat)          # <__main__.Cat object at 0x7b71d4356a10>
print(repr(cat))    # <__main__.Cat object at 0x7b71d4356a10>
print(str(cat))     # <__main__.Cat object at 0x7b71d4356a10>
```

Pomocí magických metod `__repr__` a `__str__` můžeme změnit
chování vestavěných funkcí `repr()` a `str()`.

```python
class Cat(Mammal):
  def __str__(self) -> str:
    return self.__class__.__name__ + ': ' + self.name
  def __repr__(self) -> str:
    return str(self) + ' / ' + self.sound()
  def sound(self) -> str:
    return 'miaow'

cat = Cat('Mieze')
print(cat)
print(repr(cat))
```

Print things:

```python
things: list[Thing] = [Machine('M'), Robot('R'), Cat('Miezekatze'), Dog('Athos')]

for thing in things:
  print(thing)

# <__main__.Machine object at 0x7be526652950>
# <__main__.Robot object at 0x7be526651790>
# Cat: Miezekatze
# <__main__.Dog object at 0x7be5266503d0>
```

## Vlastní iterátor

Python intenzivně používá iterátory pro generování a průchod
kolekcemi. Iterátor je objekt, který implementuje magické
metody `__iter__` a `__next__`.

Fibonacci (oh, not again!):

```python
class Fib:
  def __init__(self, limit=None):
    self.a = 0
    self.b = 1
    self.limit = limit

  def __iter__(self):
    return self

  def __next__(self):
    if self.limit is not None and self.a > self.limit:
      raise StopIteration

    value = self.a
    self.a, self.b = self.b, self.a + self.b
    return value
```

Použití:

```python
f = Fib(200)  # object
f = iter(f)   # iterator
next(f)       # iterate
```

V cyklu:

```python
for n in Fib(100):
  print(n)

for i,n in enumerate(Fib(100)):
  print(i, n)
```
