## Vyhledání prvku

### Lineární vyhledávání

Algoritmus se složitostí O(n), pracuje nad neseřazeným
seznamem. Vrací index prvního výskytu prvku v seznamu, nebo
`None` pokud prvek není nalezen.

```python
l = [1,4,3,2,9,1,8,7,4,3,5]

def lin_search(lst: list, item) -> int:
  for i in range(len(lst)):
    if lst[i] == item:
      return i
  return None

print(lin_search(l, 4))
```

Více pythonovská verze:

```python
def lin_search(lst: list, item) -> int:
  for i,val in enumerate(lst):
    if val == item:
      return i
  return None
```

### Binární vyhledávání

Vyhledáme postupným půlením intervalu (předpokládáme, že
seznam je seřazený). Složizost O(log n).

```python
def bin_search(lst: list, item) -> int|None:
  # interval <lo, hi) where item is located
  lo = 0
  hi = len(lst)

  if 0 == hi: return None # empty list

  while lo < hi-1: # while the interval has more than 1 element
    mid = (lo + hi) // 2
    if item < lst[mid]:
      hi = mid
    else:
      lo = mid

  return lo if item == lst[lo] else None

# test
lst = [1,2,4,8,16,32,64]

for n in range(-10,100):
  pos = bin_search(lst,n)
  if not pos is None:
    print(n, ':', pos)
```

### Skokové vyhledávání

Probíhá nad seřazeným seznamem, složitost O(sqrt(n)). Nejdříve
najde blok o délce _sqrt(n)_, ve kterém se prvek pravděpodobně
nachází, a poté v tomto bloku provede lineární vyhledávání.

```python
def jump_search(lst:list, item):
  lgt = len(lst)
  step = int(lgt ** .5) # floor(square root)

  # the block head and tail
  head = 0
  tail = step

  # find the corresponding block
  while tail < lgt and lst[tail] <= item:
    head, tail = tail, tail + step

  # linear search within the block
  for i in range(head, min(tail, lgt)):
    if item == lst[i]:
      return i  # found

  return None  # not found
```

### Exponenciální vyhledávání

Nad seřazeným seznamem. Je podobné skokovému vyhledávání, ale
délka bloku roste exponenciálně (2, 4, 8, 16, ...). V bloku se
pak provede binární vyhledávání.

```python
def exp_search(lst: list, item):
  lgt = len(lst)
  head = 1

  # find the block
  while head < lgt:
    if item < lst[head]:
      break
    head *= 2

  # COMPLETE:
  # the block is now lst[head//2 : min(lgt,head)]
  # search within using bin_search
```

```python
def findSubs(s: str, sub: str) -> int|None:
  if not s or not sub: return None

  for i in range(len(s) - len(sub) + 1):
    for j in range(len(sub)):
      if s[i+j]!=sub[j]:
        break
    else:
      return i

  return None

```

## Generátorové výrazy (list comprehensions)

Nahrazují klasické smyčky pro vytváření kolekcí. Generátorové
výrazy jsou čitelnější a často efektivnější.

Syntaxe: _expression_ `for` _item_ `in` _iterable_ [ `if`
*condition* ]

Odpovídající klasická smyčka pro seznam:

```python
result = []
for item in iterable:
  if condition:
    result.append(expression)
```

Uzavřené:

- v hranatých závorkách vytváří seznam
- v složených závorkách množinu nebo slovník
- v kulatých závorkách _generátor_ (nikoli n-tici)

```python
gen_list = [i for i in range(10) if i%3]
gen_set  = {i for i in range(10) if i%3}
gen_dict = {i: str(i) for i in range(10) if i%3}
gen_gen  = (i for i in range(10) if i%3)

print(gen_list)
print(gen_set)
print(gen_dict)
print(gen_gen)
```

## Vyhledání k-extrémů

Navrhněte a iplementujte algoritmus, který nalezne _k_
(k >= 1) největších prvků v seznamu.

```python
from typing import TypeAlias
Idx: TypeAlias = int
Val: TypeAlias = int|float

def k_max(lst: list[Val], k=1) -> list[tuple[Val,Idx]]|None:
  '''Return k max. items as a sorted list of tuples (value,index)'''

  if not lst or k<1: return None    # ensure correct data

  # result, with up to k candidates from the list front
  res = [(val,i) for i,val in enumerate(lst[:k])]

  # process the rest of the list
  minMax = min(res)                 # minimal (the worst) candidate
  for i,val in enumerate(lst[k:]):
    item = (val,i+k)
    if minMax < item:               # if item is a better candidate,
      res[res.index(minMax)] = item # replace the worst candidate
      minMax = min(res)             # the new worst candidate

  return sorted(res, reverse=True)  # sorted result

# test
lst: list[Val] = [2, 3, 5, 1, 6, 7, 0, 30]
k_max(lst, 4)
```
