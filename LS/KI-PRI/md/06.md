# Doporučené vývojové prostředí: Docker, VSCode, Chrome

Jako pracovní vývojové prostředí použijeme virtualizační
software
[Docker](<https://en.wikipedia.org/wiki/Docker_(software)>) a
IDE
[VS Code](https://en.wikipedia.org/wiki/Visual_Studio_Code).
Vývojové prostředí tak bude stejné (téměř), nezávisle na
operačním systému, a bude velmi podobné skutečnému webovému
serveru.

## Docker

[Docker](https://www.docker.com/) je „virtualizační
platforma“, která umožňuje vytvářet tzv. docker obrazy
(_images_) a spouštět je v docker kontejnerech (_containers_).
Obraz si lze představit jako read-only momentku (_snapshot_)
virtualizovaného počítače s operačním systémem, na kterém jsou
nainstalovány potřebné programy.

Obraz je typicky založen na minimální spustitelné verzi
operačního systému. To je tzv. základ (_base_) docker obrazu.
Potřebný obraz je pak sestaven tak, že do základu se pomocí
skriptu, uloženým v textovém souboru _Dockerfile_, doinstalují
další potřebné komponenty a programy.

Z docker obrazů lze pak spouštět nezávislé docker kontejnery,
to jest běžící virtualizované počítače, které transparentně
využívají služby hostitelského (_host_) operačního systému.

_Docker Compose_ propojí několik kontejnerů do jedné běžící
aplikace.

Pokud již nemáte, nainstalujte si
[Docker Desktop](https://docs.docker.com/engine/install/).
Tento balíček obsahuje všechny potřebné části softwaru:
_Docker Engine_ a _Docker Compose_.

## Visual Studio Code (editor)

Pokud již nemáte, nainstalujte si
[VSCode](https://code.visualstudio.com/). Můžete používat i
jiný editor zdrojového kódu, nicméně VS Code je jistý dnešní
standard, a je tedy praktické se jej naučit používat. Existuje
i komunitní verze [VSCodium](https://vscodium.com/) a další
alternativy (klony VSCode).

### Doporučená nastavení:

- _Files: Auto Save / onWindowChange_ – usnadní vám práci:
  změněné texty se budou ukládat automaticky při přepnutí do
  jiného okna.
- _Files: Trim Trailing Whitespace_ – při ukládání souboru se
  odstraní neviditelné mezery na konci řádků.
- _Editor: Format On Save_ - při uložení souboru se soubor
  zformátuje, pokud je pro něj (podle typu souboru) k
  dispozici formátovač (_Format Document With ..._)

### Doporučená rozšíření:

- [Intelephense](https://marketplace.visualstudio.com/items?itemName=bmewburn.vscode-intelephense-client)
  – inteligentní doplňování kódu pro PHP.
- [PHP Debug](https://marketplace.visualstudio.com/items?itemName=xdebug.php-debug)
  – ladění PHP kódu.
- [Live Server](https://marketplace.visualstudio.com/items?itemName=ritwickdey.LiveServer)
  – automatické obnovení prohlížeče při změně HTML, CSS, JS,
  PHP kódu.

Navíc (užitečné): editorconfig, prettier.

## Chrome (klient) nebo podobný prohlížeč (Blisk)

### Doporučená rozšíření:

- [Live Server](https://chromewebstore.google.com/detail/live-server-web-extension/fiegdmejfepffgpnejdinekhfieaogmj)

  - Actual Server Address: http://localhost:8080

# [Šablona projektu](../full-project/)

Projekt má tuto strukturu:

```bash
├── .vscode/             # konfigurace VSCode
├── docker-compose.yml   # multi-kontejner - web server a Postgres
├── docker-compose.yml   # multi-kontejner - web server a Postgres
├── docker/              # pomocné soubory pro Docker
│   ├── WebServer        # Dockerfile: Linux s Apache a PHP
│   ├── apache2.conf     # konfigurační soubor Apache
│   └── 000-default.conf # konfigurační soubor Apache
├── init-db/             # inicializace databáze
│   └── ...              # ...
└── www/                 # kořenový adresář webových stránek
     ├── .htaccess       # konfigurace Apache (per-directory)
     └── index.php       # hlavní webová stránka
```

Celou sestavu spustíme pomocí
[příkazu v terminálu](https://docs.docker.com/engine/reference/commandline/compose_up/):

```bash
docker compose up
```

_Docker Compose_ sestaví obrazy a spustí podle nich kombinaci
kontejnerů. Pokud vše proběhne správně, bude webová aplikace
přístupná v prohlížeči na URL
[http://localhost:8080](http://localhost:8000).

K běžícímu kontejneru lze připojení terminál (shell):

```bash
# vypíše běžící kontejnery
docker ps
# v kontejneru spustí interaktivní shell
docker exec -ti <container name or hash> bash
```

Další řádkové příkazy pro práci s Dockerem:

```bash
docker stop $(docker ps -q)             # zastaví běžící kontejnery
docker system prune -af --volumes       # smaže všechny kontejnery, obrazy a nepoužité disky
docker volume rm $(docker volume ls -q) # smaže všechny disky
docker builder prune --all -f           # smaže záznamy o sestavení obrazů
```

Nebo použijte grafické rozhraní
[Docker Desktop](https://www.docker.com/products/docker-desktop).
