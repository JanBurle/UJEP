# Domény a systém doménových jmen (DNS - Domain Name System)

[DNS](https://grokipedia.com/page/Domain_Name_System) je
hierarchický, distribuovaný systém, který překládá názvy (a
další prvky) domén na IP adresy (a další hodnoty). (Původ a
vznik: ARPANET, 1983)

Nejnižší úroveň (původní řešení: hosts.txt): `/etc/hosts`

Stromová hierarchie, hierarchický jmenný prostor, rozdělený do
zón:

_Root > TLDs (Top-Level Domains) > subdomains > hostnames_

### Nameservers (NS):

1. kořenové servery: obsluhují dotazy na top-level domény
   (prvního řádu / úrovně)
1. TLD servery obsluhují dotazy na domény druhého řádu (second
   level)
1. nižší servery dotazy na subdomény

Principy:

- redundance
- kešování podle TTL (time-to-live)
- dotazy přes UDP nebo TCP, port 53
- delegace autority pro nižší úrovně
- autoritativní odpovědi kryptograficky podepsané

## TLDs (Top-Level Domains)

- spravuje je [ICANN](icann.org)

### Obecné (generic) TLDs

Dnes > 1500, původních "klasických sedm":

- `.arpa` - infrastrukturní doména pro ARPANET
- `.com` - komerční entity, nejpočetnější
- `.edu` - U.S. vzdělávácí instituce
- `.gov` - U.S. vládní instituce
- `.mil` - U.S. ozbrojené síly
- `.org` - neziskové organizace
- `.net` - network infrastructure providers
- `.int` - mezinárodní organizace

V r. 2026 vznikají další nové TLDs.

### Kódy zemí a oblastí (country-codes, territories) TLDs

První byly: `.us`, `.uk`, `.ca`, `.fr`, `.de`, dnes > 300.

## Hierarchický jmenný prostor

Identifikátory oddělěné `.` čteme zprava doleva,
case-nonsensitive. Nepojmenovaný (null) kořen `.`

Typicky: `domain.tld.`, `subdomain.domain.tld.`

Celé jméno: délka < 255, dnes Unicode.

## Name Servers (NS), zóny

Autority spravují zóny a provozují jmenné servery
(nameservers, NS). Authoritativní NS spravují veškeré aktuální
záznamy pro příslušné zóny. Pro každou zónu existují alespoň
dva NS kvůli redundanci. NS vyšších úrovní neukládají data pro
podzóny, ale autoritu delegují.

- [ICANN](https://www.icann.org/en/beginners)
- [IANA](https://cs.wikipedia.org/wiki/IANA)

[Kořenový serverový systém](https://cs.wikipedia.org/wiki/Domain_Name_System#Root_servery)
má 13 NS, spravovaný 12ti organizacemi (celkem ~2000 serverů -
škálování, redundance), delegují správu domén na TLD
autoritativní servery.

## Address Resolution Process

Prohlížeč - jméno domény => local resolver (DNS client) =>
dotaz (nekešovaný) přepošle resolveru ISP (internet service
provider) nebo otevřenému NS, např.

- `8.8.8.8` (Google)
- `1.1.1.1` (Cloudflare)

(tracepath 8.8.8.8)

Hledá se v místní keši (local cache, přítomnost záznamu + TTL
time-to-live), podle potřeby se pokračuje nahoru stromem
resolverů.

## Typy záznamů v NS

- A (IPv4)
- AAAA (IPv6)
- CNAME (canonical name)
- MX (mail exchange)
- NS (name server)
- TXT (text)
- PTR (pointer, reverse lookup)

CLI nástroje:

- `nslookup` (deprecated)
- `host` (simple)
- `dig` (DNS lookup utility)

Příklady:

```bash
dig ujep.cz # SERVER: 127.0.0.53#53  A record
dig @8.8.8.8 ujep.cz
dig ujep.cz A # A AAAA MX NS TXT CNAME
dig ujep.cz CNAME # canonical name
dig ujep.cz NS # name servers
dig ujep.cz SOA # start of authority
dig -x 195.113.141.132 # reverse lookup
```

```bash
dig rqa.cz
dig -x 95.111.226.156
```

```bash
dig +trace ujep.cz
dig +trace pm.me
```

```bash
host ujep.cz
host -t A ujep.cz
```

```bash
dig +short ujep.cz
dig -x $(dig +short ujep.cz)
```

klíčovou infrastrukturu.

Geopolitická napětí: např. po ruské invazi na Ukrajinu v roce
2022 žádala Ukrajina ICANN o zrušení `.ru` a `.su` (ICANN
odmítl)

### Takedowns

Zabavování domén (domain name takedowns) - domény jsou rušeny
nebo zabavovány kvůli porušení zákona, na základě soudního
příkazu (např. kvůli nárokům na autorská práva, obchodní
známku / cybersquatting, apod.)

Slavné případy:

- [thepiratebay](https://en.wikipedia.org/wiki/The_Pirate_Bay_raid).org/.se/.sx/.ac/.pe
- megaupload.com (Kim Dotcom)
- Napster
- panavision.com
- nissan.com (neúspěšný pokus o zabavení)
- mikerowesoft.com, lindows.com
- madonna.com
