# Značkovací jazyk XML

[E**x**tensible **M**arkup **L**anguage (XML)](https://en.wikipedia.org/wiki/Xml)
je
[značkovací jazyk](https://en.wikipedia.org/wiki/Markup_language)
(a formát souboru) pro ukládání (konverzi do XML), přenos
(spuboru XML) a rekonstrukci (zpětnou konverzi) dat. Základní
principy XML jsou:

- **jednoduchost** – formát je čitelný a zpracovatelný
  člověkem i strojem (human-readable / machine-readable)
- **obecnost** – použitelný pro různé účely, tj. reprezentace
  různých datových struktur
- **textový formát** – editovatelný v textových editorech
- **univerzálnost** – podpora různých světových jazyků a
  kódování znaků
- **rozšiřitelnost** – možnost definovat vlastní značky a
  struktury

XML je založen na
[SGML](https://en.wikipedia.org/wiki/Standard_Generalized_Markup_Language)
(podobně jako HTML). XML značky (tags) reprezentují strukturu
dat a obsahují metadata (data o datech). Uvnitř značek jsou
vlastní data.

Existují mnohé knihovny a API pro zpracování XML.

Úvod do XML a ukázka:
[W3Schools XML Introduction](https://w3schools.com/xml/xml_whatis.asp).

## Struktura XML

XML kód je sestaven z _elementů_. Element je uzavřen mezi
počáteční a koncovou značkou (značky jsou součástí elementu).
Element může obsahovat atributy (v počáteční značce), text a
další (vnořené) elementy.

[W3Schools XML Elements](https://w3schools.com/xml/xml_elements.asp)

Atributy obsahují data, která se k elementu přímo vztahují:
[W3Schools XML Attributes](https://w3schools.com/xml/xml_attributes.asp).

Data uvnitř značek jsou _obsah_ (content) elementu. Může to
být text, vnořené elementy (děti – children), nebo kombinace
obou. Je otázkou , která data mají být uložena v atributech a
která v elementu. V XML neexistuje standard, který by toto
určoval, je to na rozhodnutí návrháře. Obecně se ukládají
jednoduchá data, přímo spojená s elementem, do atributů, a
více komplexní nebo hierarchická data do elementů.

[W3Schools Elements vs. Attr](http://w3schools.com/xml/xml_dtd_el_vs_attr.asp)

Názvy elementů (značek) je možné vložit do jmenných prostorů
(_namespaces_). Jmenné prostory (zapsané jako prefix značek)
jsou užitečné pro rozlišení elementů se stejným názvem, ale
jiným významem.

[W3Schools XML Namespaces](https://w3schools.com/xml/xml_namespaces.asp)

## Well-Formed (správně vytvořený, dobře strukturovaný) XML

XML je velice jednoduchý formát, který podléhá několika
jednoduchým pravidlům
[XML syntaxe](https://w3schools.com/xml/xml_syntax.asp). Pokud
jsou dodržena, pak je XML dokument tzv. dobře strukturovaný
(_well-formed_):

1. XML dokument musí obsahovat právě jeden kořenový
   (rodičovský) element, který nemá sourozence (_siblings_),
   pouze děti (_children_).
2. Pokud je v XML dokumentu prolog
   (`<?xml version="1.0" encoding="UTF-8"?>`), pak musí být
   prvním řádkem souboru.
3. Všechny elementy musí být uzavřené (výjimku tvoří prolog -
   není to element).
4. Značky jsou _case sensitive_ (na rozdíl od HTML).
5. Elementy musí být řádně zanořené, nesmí se křížit.
6. Hodnoty atributů musí být v uvozovkách, dvojitých nebo
   jednoduchých.
7. Některé znaky (`<`, `&`) mají speciální význam a proto musí
   být vloženy jako entity (`&lt;`, `&amp;`).
8. Komentáře (`<!--` ... `-->`) nesmí obsahovat dvě pomlčky
   jinde, než na začátku a na konci komentáře.
9. Bílé znaky nejsou ořezávány, na rozdíl od HTML.
10. Přechod na nový řádek je pomocí znaku LF (_line feed_) –
    na to je nutné dát pozor při práci ve Windows.

Zda je XML _well-formed_ lze otestovat pomocí XML validátorů,
např.:
[W3Schools XML Validator](https://w3schools.com/xml/xml_validator.asp),
nebo PHP validátorem.

### Příklad: Well-formed XML

Opravte následující XML kód tak, aby byl well-formed. Výsledek
otestujte validátorem.

#### `knihy.xml`

```xml
<kniha>
    <!-- každá kniha obsahuje dva názvy a to český a anglický -- specifikováno atributem -->
    <název jazyk=cz>Epos o Berygamešovi
    <název jazyk=en>Epic of Berygamesh
    <Autor>Jiří Fišer</autor>
    <postavy><postava>Berygameš</postava>
    <postava>Škvorkidu<postavy/></postava>
</kniha>
<kniha>
    <název jazyk=cz>Pán prstenů: návrat Fišera
    <název jazyk=en>Lord of the rings: return of Fišer
    <Autor>Beránek Pavel</autor>
    <popis>
        Kniha o partě ajťáků, kteří se chystají na výpravu na zápočet na Fakultu Osudu.
    </popis>
</kniha>
<?xml version="1.0" encoding="UTF-8"?>
```

## Validní XML

Nestačí, aby byl XML soubor dobře strukturovaný. Dobře
strukturovaný XML by měl být také validní.

Struktura validního XML dokumentu musí odpovídat šabloně ve
formátu **DTD** (_Document Type Definition_) nebo **XSD** (XML
Schema Definition, novější typ šablony, která je sama o sobě
ve formátu XML).

Ukázka DTD:
[W3Schools XML DTD](https://w3schools.com/xml/xml_dtd.asp).

Z hlediska DTD se XML dokument se skládá ze stavebních bloků:
[W3Schools XML DTD Building Blocks](https://w3schools.com/xml/xml_dtd_building.asp).
DTD definuje jak na sebe bloky navazují – jaké každý blok
obsahuje potomky a jaké má příbuzné.

Stavební bloky XML z hlediska DTD: elementy, atributy, entity,
parsovaná data (PCDATA), znaková data (CDATA)

## XML služby

_Zastarávající technologie: jejich používání klesá, ale v
některých prostředích jsou stále relevantní: podnikové
systémy, bankovnictví, apod._

XML se používá pro předávání dat mezi servery, které poskytují
služby, a klientskými aplikacemi, které tyto služby využívají.
Mezi základní XML formáty, které jsou používány pro
[XML služby](https://www.w3schools.com/xml/xml_services.asp),
patří:

- [XML WSDL](https://w3schools.com/xml/xml_wsdl.asp) (_Web
  Services Description Language_) – popisuje webovou službu,
  kde ji najít a jak se k ní připojit, jaké datové typy
  používá ve zprávách, jaké operace poskytuje, a jaký je
  protokol (posloupnost) komunikace. WSDL se používá společně
  se SOAP.
- [XML SOAP](https://w3schools.com/xml/xml_soap.asp) (_Simple
  Object Access Protocol_) – formát a protokol pro výměnu dat
  mezi systémy a volání vzdálených procedur pomocí HTTP
  požadavků, čistě v XML (na rozdíl od jiných typů podobných
  služeb, jako je např.
  [CORBA](https://en.wikipedia.org/wiki/Common_Object_Request_Broker_Architecture)).
- [XML RDF](https://w3schools.com/xml/xml_rdf.asp) (_Resource
  Description Framework_) – formát pro popis webových metadat
  (pro tzv. sémantický web).
- [XML RSS](https://w3schools.com/xml/xml_rss.asp) (_Really
  Simple Syndication_) – formát pro syndikaci webového obsahu
  – pro zasílání krátkých upozornění odběratelům (typicky do
  aplikací typu RSS čtečka).

Více na:

- tutorialspoint.com:
  [WSDL](https://www.tutorialspoint.com/wsdl/),
  [SOAP](https://www.tutorialspoint.com/soap/),
  [RSS](https://www.tutorialspoint.com/rss/)
- [ukázka WDSL](https://en.wikipedia.org/wiki/Web_Services_Description_Language)
- více o SOAP :
  [guru99.com](https://guru99.com/soap-simple-object-access-protocol.html)

  [linkeddatatools.com](https://linkeddatatools.com/semantic-web-basics)

## Jazyk XSD

XSD (XML Schema Definiton) je alternativa k šabloně v DTD
(document type definition) formátu. Tento způsob psaní schémat
je složitější, ale i mocnější, než DTD.

- Popis [XML Schema](https://w3schools.com/xml/xml_schema.asp)
- Porovnání XSD a DTD schémat:
  [XML Schema How To](https://w3schools.com/xml/schema_howto.asp)

### XSD dokument

Základní stavební bloky XSD dokumentu jsou:

1. Kořen
   [XSD &lt;xs:schema>](https://w3schools.com/xml/schema_schema.asp)
   – každý XSD potřebuje kořenový element, v něm se mohou
   nacházet definice jmenných prostorů.
2. Elementy – jsou dvou typů a to _simple_ a _complex_;
   [XSD Simple Elements](https://w3schools.com/xml/schema_simple.asp)
   obsahují data v primitivních datových typech jako jsou
   string, decimal, integer, boolean, date a time;
   [XSD Complex Elements](https://w3schools.com/xml/schema_complex.asp)
   obsahují další elementy nebo atributy.
3. Atributy
   [XSD Attributes](https://w3schools.com/xml/schema_simple_attributes.asp)
   – obsahují data stejných typů jako simple elementy a mohou
   mít implicitní (default), pevně dané (fixed) nebo
   vyžadované (required) hodnoty, a dělají z elementu
   komplexní typ.
4. Restrikce (omezení)
   [XSD Restrictions](https://w3schools.com/xml/schema_facets.asp)
   – určují rozsah nebo výčet akceptovaných hodnot pro
   elementy nebo atributy.
5. Data – jsou typu
   [string](https://w3schools.com/xml/schema_dtypes_string.asp),
   [date](https://w3schools.com/xml/schema_dtypes_date.asp),
   [numeric](https://w3schools.com/xml/schema_dtypes_numeric.asp),
   [misc](https://w3schools.com/xml/schema_dtypes_misc.asp)
   (boolean, binary, anyURI, double, float, atd.)

### XSD Complex elements

Komplexní element
[XSD Complex Elements](https://w3schools.com/xml/schema_complex.asp)
je element, který obsahuje další elementy, atribut a/nebo
restrikce. Komplexní elementy jsou čtyř druhů:

1. Prázdné elementy
   [XSD Empty Elements](https://w3schools.com/xml/schema_complex_empty.asp)
   – elementy bez obsahu, které ale mohou mít atributy (takže
   mohou obsahovat data).
2. Pouze-elementy
   [XSD Elements Only](https://w3schools.com/xml/schema_complex_elements.asp)
   – element, který obsahuje pouze další elementy.
3. Pouze-text
   [XSD Text Only Elements](https://w3schools.com/xml/schema_complex_text.asp)
   – element který obsahuje text a atributy.
4. Smíšené
   [XSD Mixed Content](https://w3schools.com/xml/schema_complex_mixed.asp)
   – kombinace předchozích.

### XSD Indikátory

Jazyk XSD obsahuje mechanismus indikátorů
[XSD Indicators](https://w3schools.com/xml/schema_complex_indicators.asp),
které jsou určené k tomu, aby řídily způsob používání XML
elementů. Dělíme je na indikátory:

1. Řádu – řídí pořadí elementů (all = libovolné pořadí, choice
   = exkluzivní výběr, sequence = přesně dané pořadí).
2. Výskytu – řídí počet elementů (maxOccurs = maximální počet
   elementů, minOccurs = minimální počet elementů).
3. Skupiny – řídí uspořádání prvků do pojmenovaných skupin.

### Substituční skupiny

Další užitečným mechanismem je substituční mechanismus
[XSD Element Substitution](https://w3schools.com/xml/schema_complex_subst.asp),
který lze využít např. pro vícejazyčné XML dokumenty. XSD
umožňuje definovat skupinu zaměnitelných elementů (tzv.
substituční skupina), ve které jsou elementy vzájemně
zaměnitelné.

### XSD Libovolné elementy

Pokud chceme ještě větší volnost a umožnit vložit uživateli v
XML v nějaké části libovolné elementy, pak můžeme využít v XSD
element <any>
[XSD Any Element](https://w3schools.com/xml/schema_complex_any.asp).
Obdobný mechanismus existuje i pro atributy elementů pomocí
<anyAttribute>
[XSD anyAttribute](https://w3schools.com/xml/schema_complex_anyattribute.asp).

# Zobrazení XML pomocí CSS a XSLT

## Zobrazení XML v prohlížeči

## Seznámení s XSLT a XPath

Validní XML data lze transformovat do HTML (a jiných formátů)
pomocí jazyka XSLT (eXtensible Stylesheet Language
Transformations). Příklad XSLT naleznete na
[XML XSLT](https://w3schools.com/xml/xml_xslt.asp) a tutoriál
na
[XSLT Introduction](https://www.w3schools.com/xml/xsl_intro.asp).

Pro navigaci v XML dokumentu v XSLT slouží jeho komponent
[XPath](https://www.w3schools.com/xml/xml_xpath.asp). Tutoriál
je na
[XPath Tutorial](https://www.w3schools.com/xml/xpath_intro.asp).

## Převod XML do HTML pomocí XSLT

Ukázka transformace z XML do HTML je na stránce
[XSLT Transformation](https://w3schools.com/xml/xsl_transformation.asp).
Transformace se zde provádí především pomocí XSLT elementů:

- [xsl:template](https://www.w3schools.com/xml/xsl_templates.asp)
  (šablona)
- [xsl:for-each](https://w3schools.com/xml/xsl_for_each.asp) -
  realizace cyklu v XSLT z vyfiltrovaného výběru XML uzlů
- [xsl:value-of](https://w3schools.com/xml/xsl_value_of.asp) -
  získá data z jednoho uzlu a využije je při transformaci

Příklad: soubor `cdcatalog.xsl` Příklad: soubor `student.xsl`

### Podmínky a cykly v XSLT

V jazyce XSLT lze využívat podmínky a cykly, které řídí, na
které uzly a jak aplikovat pravidla v šabloně, pomocí dalších
elementů:

- [xsl:if](https://w3schools.com/xml/xsl_if.asp) - slouží jako
  realizace podmínky v XSLT
- [xsl:choose](https://w3schools.com/xml/xsl_choose.asp) -
  realizace přepínače v XSLT
- [xsl:sort](https://w3schools.com/xml/xsl_sort.asp) - slouží

### Využití XPath v XSLT

Jazykem XPath volíte v XSL uzly nebo množiny uzlů. Projděte si

## Použití XML/SimpleXML pro ukládání dat

XML soubory lze použít pro ukládání dat, místo databáze.
Pomocí SimpleXML lze z dat vytvořit XML strom a ten pak na
serveru uložit do souboru ve vhodně zvoleném adresáři, který
není přístupný pomocí HTTP, ale jen z PHP kódu. XML soubory
lze pak načítat, upravovat a znovu ukládat.
