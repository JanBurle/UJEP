# Funkce 🠊 struktura programu

## Definice, deklarace a volání funkcí

Funkce musí být deklarována nebo definována před voláním
(dříve: implicitní deklarace).

### Deklarace funkce (prototyp)

- informuje překladač o existenci funkce
- může se opakovat vícekrát
- umožňuje volat funkci _před její definicí_ (forward
  declaration)
- obvykle v hlavičkových souborech (`.h`)

### Definice funkce

Definice funkce:

- obsahuje úplnou implementaci funkce včetně těla
- může se vyskytovat pouze jednou v programu, jinak dojde k
  chybě při překladu nebo linkování
- obvykle v implementačních souborech (`.c`)

### Volání funkce

- založí rámec pro volání funkce (call frame) na zásobníku
- vyhodnotí argumenty a uloží je na zásobník (nebo do
  registrů)
- předá řízení do těla funkce (strojový příkaz CALL)
- při dokončení (RET) vrátí návratovou hodnotu (pokud není
  void) a obnoví původní rámec

V jazyce C (binární standard volání): rámec obnovuje
_volající_ funkce\
V jazyce Pascal: rámec obnovuje _volaná_ funkce)

## BNF

[BNF jazyka C](https://cs.wmich.edu/~gupta/teaching/cs4850/sumII06/The%20syntax%20of%20C%20in%20Backus-Naur%20form.htm):

```bnf
<function-declaration> ::= <declaration-specifiers> <declarator> ";"

<declaration-specifiers> ::= <type-specifier> | <type-qualifier> <declaration-specifiers>
<type-specifier>        ::= "void" | "char" | "int" | "float" | "double" | <typedef-name>

<declarator>            ::= <direct-declarator>
<direct-declarator>     ::= <identifier> "(" <parameter-type-list>? ")"

<parameter-type-list>   ::= <parameter-list> ["," "..."]
<parameter-list>        ::= <parameter-declaration> | <parameter-list> "," <parameter-declaration>
<parameter-declaration> ::= <declaration-specifiers> <declarator>
```

```bnf
<function-definition> ::= <declaration-specifiers> <declarator> <compound-statement>

<compound-statement>  ::= "{" <block-item-list>? "}"

<block-item-list>     ::= <block-item> | <block-item-list> <block-item>
<block-item>          ::= <declaration> | <statement>

<statement>           ::= <labeled-statement> | <expression-statement>
                          | <selection-statement> | <iteration-statement>
                          | <jump-statement>
```

```bnf
<function-call>         ::= <postfix-expression> "(" <argument-expression-list>? ")"

<postfix-expression>    ::= <identifier>
                            | <postfix-expression> "[" <expression> "]"
                            | <postfix-expression> "." <identifier>

<argument-expression-list> ::= <assignment-expression>
                               | <argument-expression-list> "," <assignment-expression>
```

## Parametry, argumenty a návratové hodnoty

- **parametry**: formální proměnné v definici funkce
- **argumenty**: skutečné hodnoty předané při volání

### Předávání argumentů hodnotou (pass by value):

- vytvoří se kopie argumentu
- změny uvnitř funkce neovlivní původní proměnnou
- standardní způsob v C

#### Příklad:

```c
void inc(int x) {
    x++;
}

int a = 5;
inc(a);
// a je stále 5
```

### Předávání ukazatelem (pass by pointer/reference):

- umožňuje měnit původní proměnnou
- předává se _adresa_ proměnné

#### Příklad:

```c
void inc(int *x) {
    (*x)++;
}

int a = 5;
inc(&a);  // předáváme adresu a
// a je nyní 6
```

### Návrat a návratové hodnoty

- Klíčové slovo `return` ukončí funkci a vrátí hodnotu
- Typ návratové hodnoty musí odpovídat deklaraci
- `void` funkce nemusí mít `return` nebo může mít `return;`
  bez hodnoty
- Příklad:

```c
int add(int a, int b) {
    return a + b;
}

void say_hello() {
  printf("Hello, World!\n");
}

```

### Předčasný návrat (early return)

- Funkce může mít více `return` příkazů
- Příklad:

```c
int max(int a, int b) {
    if (a > b) return a;
    return b;
}
```

## Rozsah platnosti proměnných (scope) a doba života (lifetime)

### Lokální proměnné (automatické)

- deklarované uvnitř funkce nebo bloku
- viditelné pouze v bloku, kde jsou deklarované
- existují pouze během vykonávání bloku
- uložené na zásobníku (stack)
- každé volání (sekvenční i rekurzivní) funkce má vlastní
  kopii
- bez inicializace obsahují nedefinované hodnoty

### Globální proměnné

- deklarované mimo funkce
- viditelné ve všech funkcích od místa deklarace dále
- existují po celou dobu běhu programu
- automaticky _inicializované na 0_ (linker)
- uložené v datové sekci paměti
- používat opatrně a střídmě (v C++ je problematická sekvence
  inicializace)

### Externí proměnné

- deklarované s klíčovým slovem `extern`
- deklarace bez definice (nealokují paměť)
- umožňují sdílet globální proměnné mezi více soubory
- v jednom souboru musí být definice (bez `extern`)

### Statické proměnné

- **lokální statické**: `static int x = 0;`
  - viditelnost: uvnitř funkce
  - doba života: celý běh programu
  - zachovávají hodnotu mezi voláními
  - inicializují se pouze jednou, při první inicializaci

- **globální statické**:
  - viditelnost omezena na aktuální překladovou jednotku =
    `.c` soubor (translation unit)
  - uložené v datové sekci paměti

### Automatické proměnné

- deklarované bez `static` uvnitř funkce
- viditelné pouze v bloku, kde jsou deklarované
- existují pouze během vykonávání bloku
- uložené na zásobníku (stack)
- bez inicializace obsahují ne definované hodnoty

#### Registerové proměnné

- automatické proměnné deklarované s klíčovým slovem
  `register`
- návrh pro uložení v registru (není zaručeno)
- nemají adresu, nelze použít `&`
- dnes je optimalizace ponechána překladači

### Dynamické proměnné

- alokované pomocí `malloc()`, `calloc()`, `realloc()`
- existují od alokace do uvolnění (`free()`)
- uložené na haldě (heap)
- musí být explicitně uvolněné, jinak dochází k úniku paměti
  (memory leak)

## Shrnutí

### Rozsah platnosti/viditelnosti (scope)

- **blok**: proměnná platí pouze v bloku { }
- **funkce**: parametry a lokální proměnné funkce
- **soubor, program**: statické a globální proměnné

### Doba života (lifetime)

- **automatická**: od deklarace do konce bloku
- **statická**: celý běh programu
- **dynamická**: od `malloc()` do `free()`

# Modulární struktura programu

- rozdělení programu do více souborů / modulů / překladových
  jednotek
- každý modul může mít svoje rozhraní
- **hlavičkový soubor** (.h): deklarace globálních typů,
  funkcí, globálních proměnných
- **implementační soubor** (.c): definice lokálních typů,
  funkcí a proměnných (static)

# Rekurze

- funkce volá sama sebe
- měla by mít ukončovací podmínku
- každé rekurzivní volání má vlastní rámec na zásobníku s
  kopiemi lokálních proměnných

#### Příklad:

```c
int fact(int n) {
    if (n <= 1) return 1;   // ukončovací podmínka
    return n * fact(n - 1); // rekurzivní volání
}
```

## Typy rekurze

- **přímá**: funkce volá přímo samu sebe
- **nepřímá**: např. funkce A volá B, která volá A
- **koncová** (tail recursion): rekurzivní volání je poslední
  operace ve funkci, umožňuje optimalizaci (tail call
  optimization), nikoli však v C standardu
