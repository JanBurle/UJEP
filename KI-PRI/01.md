## Programování pro Internet

### Co to je, Internet?

Globální síť počítačů propojených standardizovanými protokoly.

#### Protokoly:

Propojení _klient/server_, komunikace, přenos dat.

- [TCP/IP](https://www.geeksforgeeks.org/tcp-ip-model/)
- [HTTP](https://www.geeksforgeeks.org/http-full-form/), SSH, FTP, SFTP, SMTP, POP3, ...

### TCP/IP

Spojení (datový proud) s [IP adresa:port](https://www.geeksforgeeks.org/tcp-ip-ports-and-its-applications/).

Porovnejte s datagramy [UDP](https://www.geeksforgeeks.org/user-datagram-protocol-udp)

### HTTP (nad TCP/IP)

Také (dnes především) [HTTPS](https://www.geeksforgeeks.org/explain-working-of-https).

Typicky:

- Klient – vysílá požadavky, přijímá a zobrazuje a interpretuje HTML (XML), CSS, JS data/soubory.
- Server: přijímá požadavky, generuje a posílá data/soubory.

* Klient: nejčastěji prohlížeč
* Server: nejčastěji [LAMP](<https://en.wikipedia.org/wiki/LAMP_(software_bundle)>) (Linux, Apache, MySQL(?), PHP)

## Vývojové prostředí: Docker, VSCode, Chrome

Jako pracovní vývojové prostředí použijeme virtualizační software [Docker](<https://en.wikipedia.org/wiki/Docker_(software)>) a IDE [VS Code](https://en.wikipedia.org/wiki/Visual_Studio_Code). Vývojové prostředí tak bude stejné (téměř), nezávisle na operačním systému.

### Docker

[Docker](https://www.docker.com/) je softwarový nástroj („virtualizační platforma“), která umožňuje vytvářet tzv. docker obrazy (_images_) a spouštět je v docker kontejnerech (_containers_). Obraz si lze představit jako read-only momentku (_snapshot_) virtualizovaného počítače s operačním systémem, na kterém jsou nainstalovány potřebné programy.

Obraz je typicky založen na minimální spustitelné verzi operačního systému. To je tzv. základ (_base_) docker obrazu. Potřebný obraz je pak sestaven tak, že do základu se pomocí skriptu, uloženým v textovém souboru _Dockerfile_, doinstalují další potřebné utility a programy.

Z docker obrazů lze pak spouštět nezávislé docker kontejnery, to jest běžící virtualizované počítače, které transparentně využívají služby hostitelského (_host_) operačního systému.

_Docker Compose_ propojí několik kontejnerů do jedné běžící aplikace.

### Docker (virtuální server)

Pokud již nemáte, nainstalujte si [Docker Desktop](https://docs.docker.com/engine/install/). Tento balíček obsahuje všechny potřebné části softwaru: _Docker Engine_ a _Docker Compose_.

### Pracovní projekt: pracovní složky/adresáře a soubory

Stáhněte si [pracovní projekt](../Project1).
(Tento projekt byl základem pro cvičení v LS 2024.)

Projekt má tuto strukturu:

```bash
├── .vscode/             # konfigurace VSCode workspace
├── docker-compose.yml   # definice multi-kontejneru
├── Dockerfiles/         # soubory pro Docker
│   ├── PhpApache        # virtuální Linux s Apache a PHP
│   ├── Database         # MySQL kontejner
│   └── univerzita.sql   # SQL skript pro inicializaci databáze
└── www/                 # odpovídá /var/www serveru
     ├── html/           # kořen webových stránek (v dosahu Apache)
     │   ├── .htaccess   # per-directory konfigurace Apache
     │   └── ...         # atd.
     └── ...             # atd.
```

### Jednotlivé soubory

#### `.vscode`

Doporučená rozšíření, skrývání souborů v navigačním panelu, spouštění debuggeru.

#### `docker-compose.yml`

Nástroj _Docker Compose_ propojí několik Docker obrazů do jednoho spolupracujícího celku a spustí tzv. multikontejnerovou aplikaci.

Konfigurace aplikace je popsána v souboru `docker-compose.yml`, podle kterého Docker Compose:

1. Sestaví obraz z _Dockerfile_ PhpApache, propojí ho s databázovým obrazem, a mapuje vniřní port 80 webového serveru na vnější port 8000. Dále mapuje vnitřní adresář Apache serveru, ve kterém jsou data pro webovou stránku a jiné, na náš vnější adresář.
2. Stáhne databázový obraz [mysql](https://hub.docker.com/_/mysql) z _Docker Hub_ repositáře, nastaví root heslo, vytvoří uživatele _admin_ a nastaví mu heslo _heslo_, a inicializuje databázi.
3. Stáhne docker obraz [adminer](https://hub.docker.com/_/adminer/) z _Docker Hub_, a nastaví jeho vnější port na 8080.

#### `Dockerfiles/PhpApache` (VPS)

Stáhne základní obraz Linuxové distribuce s nainstalovaným jazykem PHP a Apache serverem. Nainstaluje aktualizace systému, které proběhly po vytvoření základního obrazu. Nainstaluje rozšíření PHP s XSL procesorem (který není součástí standardní distribuce PHP). Nainstaluje PHP rozšíření (driver) [myslqi](https://www.php.net/manual/en/book.mysqli.php) pro připojení PHP k MySQL databázi. Nainstaluje pomocné programy.

Základní obraz [php:8-apache](https://hub.docker.com/_/php) je minimální Linuxová distribuce Debian, s nainstalovaným interpretrem PHP a webovým serverem Apache ([httpd](https://en.wikipedia.org/wiki/Httpd)).

#### `Dockerfiles/Database`

Stáhne základní obraz s MySQL a inicializuje databázi ze SQL skriptu.
Eventuelně vyměňte/vyměníme za PostgreSQL.

#### `www/html/.htaccess`

V kořenovém adresáři `www/html` je soubor [.htaccess](https://httpd.apache.org/docs/current/howto/htaccess.html) ve kterém je povoleno, aby Apache generoval obsah adresáře. Nemusíme tak mít soubor `index.php` (nebo `index.html`).

#### `www/html/example.html`

Soubor `example.html` obsahuje minimální webovou stránku.

### Docker v příkazové řádce

Celou sestavu spustíme pomocí [příkazu v terminálu](https://docs.docker.com/engine/reference/commandline/compose_up/):

```bash
docker compose up
```

_Docker Compose_ sestaví obrazy a spustí podle nich kombinaci kontejnerů. Pokud vše proběhne správně, bude naše webová aplikace přístupná v prohlížeči na URL [http://localhost:8000](http://localhost:8000) a administrace databáze na [http://localhost:8080](http://localhost:8080).

K běžícímu kontejneru lze připojení terminál (shell):

```bash
docker ps                           # vypíše běžící kontejnery
docker exec -ti project-vps bash    # v kontejneru spustí interaktivní shell
```

Pokud máte `bash` jako shell (Linux, Cygwin, ...), můžete použít následující skript pro zastavení běžících kontejnerů a vymazání všech kontejnerů, obrazů a disků v Dockeru:

```bash
docker stop $(docker ps -q)         # zastaví běžící kontejnery
docker system prune -af --volumes   # smaže všechny kontejnery, obrazy a nepoužité disky
docker volume rm $(docker volume ls -q) # smaže všechny disky
```

Nebo použijte [Docker Desktop](https://www.docker.com/products/docker-desktop) s grafickým rozhraním.

## HTML, XHTML

Zdrojový kód většiny webových stránek je psaný ve značkovacím jazyce (markup language), typicky v nějaké verzi HTML:

- dnes: [HTML5](https://en.wikipedia.org/wiki/HTML5)
- v přechodném období [XHTML](https://en.wikipedia.org/wiki/XHTML).

Panuje/panoval jistý chaos v tom, co je co, také proto, že celý webový „ekosystém“ je poznamenán svým bouřlivého vývojem a s ním spojenými [„válkami prohlížečů“](https://en.wikipedia.org/wiki/Browser_wars).

Protože mnoho (většina?) webových stránek obsahuje chyby, prohlížeče mají snahu chyby v HTML tolerovat a opravovat.

Jisté vysvětlení rozdílů mezi těmito značkovacími jazyky naleznete např. [zde](https://www.w3schools.com/Html/html_xhtml.asp) nebo [zde](https://hackr.io/blog/difference-between-html-html5-xhtml).

Validitu různých formátů lze ověřit např. [W3 validátorem](https://validator.w3.org/).

## Can I use?

HTML, CSS (JS, PHP) se vyvíjí. Doporučená strategie při vývoji webových aplikací:

- příliš se netrápte tím, aby aplikace byly tzv. zpětně kompatibilní s příliš starými prohlížeči (více než dva roky)
- používejte moderní verze jazyků a technologií, i za cenu toho, že nebudete podporovat prohlížeče, které nedrží krok s dobou (zmínil tu někdo IE?)

Pro ověření, zda lze použít funkci/aspekt webových technologií: [Can I use](https://caniuse.com/ciu/index)
